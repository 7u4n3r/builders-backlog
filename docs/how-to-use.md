# How To Use Builders Backlog

Builders Backlog is a holding system for unfinished ideas, fragmented research, half-built tools, and project context.

Use it when something is real enough to keep, but not clear enough to finish yet.

## 1. Capture the fragment

Start by writing down the idea exactly as it is.

Do not clean it up too early.
Do not force it into a project plan.
Do not decide whether it is good yet.

Just capture it.

## 2. Add context

Every idea needs context or it becomes noise later.

Add:

- Where it came from
- What problem it is connected to
- Who it might help
- What triggered the idea
- What unfinished material already exists

## 3. Sort the state

Mark the idea as one of these:

- Raw fragment
- Research needed
- Build started
- Needs testing
- Ready to package
- Ready to publish
- Archive for later

## 4. Choose the next small move

Do not try to finish the whole thing at once.

Choose one next move:

- Find the missing file
- Write the first paragraph
- Make the first screen
- Create a folder
- Record a short explanation
- Turn it into a Gumroad draft
- Move it to GitHub

## 5. Review weekly

Once a week, scan the backlog and ask:

- What is still alive?
- What is dead weight?
- What can become a product?
- What can become a post?
- What needs proof?
- What needs to be deleted?

Builders Backlog is not meant to make every idea finished.

It is meant to stop usable ideas from disappearing before they have a chance to become something.
